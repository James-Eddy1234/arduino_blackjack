The goal of this project was to make a blackjack game. Instead of cards, the game uses dice. This stemmed from the original project idea, which was just rolling a dice with the goal of the rolls summing up to a randomly chosen goal amount. The switch to blackjack instead came from a recommendation comment in Netrun, as this would result in a more fair game because the original idea had an unlikely winning chance. The hardware that was used for this was a breadboard, two buttons, an OLED display, a 4 digit 7 segment display, the Arduino from the lab kit, and some wires and resistors. 

For the technical aproach to this, I would break this down into organizing the hardware and wiring and also how the code was approached. For the hardware, the OLED and both buttons are all hooked up to the ground rail on the breadboard, going to a single ground pin on the Arduino. This was to make it so that there were less wires to make organization easier. Each button is also wired to an analog pin to read when the button is pressed. The OLED is also wired to two analog pins (A4 and A5, as these are the defaults with the library) and wired to the 3.3V pin on the Arduino. For the 7 segment display, each digits ground pin has a 1k Ohm resistor, each of which goes to a digital pin. The segment pins are each wired to a digital pin as well. For the approach with the code, the pin numbers are defined first along with an array of arrays of Integers. This table is used to define number patterns for the 7 segment display. There are also a number of variables that store various values needed for the game, such as button states, roll sums, roll values, and the counters for wins, losses, and ties. There are functions for the following: rolling the dice, showing a number at a specific digit, displaying the sums for the player and dealer, drawing the dots on the dice face, and resetting the game. The function for displaying a single number works by going through the previously mentioned table with number patterns. The functions for displaying sums use an array, calculating the tens and ones places of the sums. The reset function resets roll values, dealer and player sums, button states, and displays zeros on the segment display. The setup function declares the pin modes, begins the OLED and sets the font, and chooses a random seed for rand() by doing an analog read on an empty and unused pin. The loop function uses a multiplexing method to refresh the OLED and segment display after certain time intervals. This needed to be done because the segment display was not being updated fast enough without this, and visibly flickered to the point it was almost unreadable. At the moment, the flicker is still present, but to a much lesser extent than it was originally. This method defines an unsigned long, which retains its value across loop iterations, and an int defining a refresh interval of 500. What this method does is it makes sure that there is only so much time that can pass without one of the displays from being updated. This makes the segment display get updated more frequently than it was originally, as the OLED display takes time to update, to the point that it blocks the segment display from being updated frequently enough to not visibly flicker. Additionally, there is also code checking to see if the hit or stand buttons were pressed. If the hit button is pressed, then it calls the dice rolling function to roll a dice, adding this value to rollSum. If the stand button is pressed, then a while loop is run to get the dealer's sum. Like regular blackjack, the dealer must hit until the sum is greater than 16. The same if statement is run if the player overshoots 21. After this while loop, the dealer's sum is displayed and the variable gameOver is set to true. After this if statement is another if statement that checks to see if gameOver is true. It then creates two variables that subtract the player sum from 21 and the dealer sum from 21. These are used in a chain of if, else if, and else statements that check to see if the player won, the dealer won, or if there was a tie. The order of these checks is important, as if certain conditions are true then the rest are irrelevant. The first check is if the player busted, then if the player hit 21 and the dealer did not, then if the dealer busted, then if the player was closer to 21, then if the dealer was closer to 21, otherwise it is a tie. Depending on which if statement runs, 1 is added to the win counter, the lose counter, or the tie counter. A value set equal to millis() is used to keep the if statement that runs the game reset from happening for two seconds, this is to allow the player to have the time to read their sum and the dealer sum before they are reset to 0. The win, loss, and tie counters are printed at the top of the OLED display. oled.drawFrame is used to draw the box for the dice face. A switch statement is used to check the value of rollValue, printing the dots for the correct dice face.
